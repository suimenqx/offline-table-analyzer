# Offline Table Analyzer

Offline Table Analyzer 是一个隐私优先、完全离线的表格整理与分析工作台。它以单个 `index.html` 交付，可直接处理从终端、网页、Excel、Markdown 和日志中复制出来的杂乱表格，不上传数据、不需要账号，也没有运行时依赖。

当前版本：**20.1.0**

## 项目初衷

运维数据极少以整洁的电子表格形式出现。它往往来自终端输出、浏览器、Markdown 文档、Excel 或诊断日志。Offline Table Analyzer 将这些输入转换为统一的表格模型，并提供实用的分析工具，帮助你快速完成数据处理。

- 完全离线，运行时零依赖
- 单文件 `index.html`，即开即用
- 无账号、无遥测、无分析追踪、无网络请求
- 为敏感的运行排障数据而设计
- 中文文档配合 [英文主文档](README.md)

## 功能特性

### 导入与解析

源文本经过 10 级解析器管道按优先级依次尝试：

1. **CLI table-data** — 传统多表格式，含 `table-data` 和 `validflag` 标记
2. **HTML 剪贴板表格** — 支持 `colspan`、`rowspan`、`<br>`
3. **ASCII / 终端表格**
4. **Markdown / 竖线表格** — 含转义竖线
5. **TSV / Excel 粘贴**
6. **CSV** — 引号逗号、转义引号、多行单元格
7. **分号分隔 CSV**
8. **固定宽度文本** — 通过连续空格分隔列
9. **定宽对齐文本** — 从表头字符位置推断列边界的无分隔符输出
10. **空白分隔纯文本**（兜底解析）

更多导入能力：

- **表头模式**：自动推断、强制首行表头、或生成表头（`Column1`、`Column2`…）。CLI table-data 使用 `validflag` 行作为表头。空白和重复表头自动规范化。解析诊断信息在界面中可见。
- **定宽对齐输入**：识别纯 `-` 分隔线，支持表格上下分隔线、表头与数据之间的分隔线、同一输入中的多张表，并将 `--` 单元格解释为空值。
- **数据输入**：粘贴、拖放或文件选择。根据文件扩展名自动识别格式（`.csv` / `.tsv` / `.html` / `.htm` / `.md` / `.markdown`）。
- **编辑器**：可拖拽调整高度的源文本框（120–600 px），以及用于大段输入的**全屏数据源编辑器**。
- 源文本和工作区文件均有 25 MB 安全限制。

### 分析工作台

- **多个可命名、可拖拽排序的分析页签**，每个页签独立管理状态。
- **三级筛选**：全局筛选（跨表）、按表筛选、按列筛选。规则支持等于、包含、不等于、数值比较、正则、AND/OR 逻辑组合，以及带引号的多词搜索。
- **高亮规则**与**仅显示高亮行 (Focus)** 模式，隐藏未匹配的行。
- **逐表显示列选择**与表格卡片折叠。
- **列表头预览**与**转置行表头预览**两种模式，方便审视宽表或长表。
- **分页渲染**：每表 50 / 100 / 250 / 500 行可选。
- **可持久化的单元格修订**，支持撤销/重做（`Ctrl`+`Z` / `Ctrl`+`Y`）。
- 当源文本、导入文件、解析格式或表头模式变化时，旧修订会在界面提示下清除，避免按行保存的修订误应用到其他记录。

### JOIN 视图

- 六种连接类型：**Inner**、**Left**、**Right**、**Full**、**Semi**、**Anti**。
- 支持**多条等值条件**。
- 左右两侧均可选择原始解析表或已有的视图（支持链式 JOIN）。
- **列搜索**、**别名**（`列名 AS 别名` 或 `列名: 别名`）以及**拖拽重排**输出列。
- **自动匹配**左右两侧同名列。
- **依赖循环检测**，防止视图间形成循环引用。
- 设计过程中实时展示**匹配/未匹配行数估算**。
- **防冲突复合键**，正确处理 `0` 和 `false` 值。

### 复制、导出与恢复

- **复制格式**：TSV（默认）、CSV、Markdown、ASCII。每次复制同时附带 HTML 剪贴板内容，方便粘贴到富文本编辑器中。
- **电子表格公式注入保护**：以 `=`、`+`、`@` 或非数字 `-` 开头的单元格内容会添加前缀，避免粘贴到电子表格软件后被意外执行。
- **Excel 导出**：
  - **原始 Excel** — 解析后的原始表格，不含任何过滤。
  - **全量 Excel** — 选择要导出的表和视图，支持按当前显示列投射。
  - **预览 Excel** — 当前筛选和分页后的结果。
- **安全数字序列化**：超过 15 位有效数字、带不安全前导零、高精度数字样式字符串以及非有限数值会按 Excel 文本导出，以保留标识符和精度。
- **版本化工作区备份**（JSON）：导出整个工作区，恢复时可选择**替换**当前工作区或**追加为新的分析页签**。
- **配置导出/导入**：仅含规则和视图，不含原始数据，限 5 MB。适合分享分析设置而不暴露底层数据。
- **临时数据模式**：关闭原始文本持久化后，源数据仅保留在当前页面会话中；分析规则和界面偏好仍会保存。
- **状态栏存储状态**：显示存储用量条，`localStorage` 配额耗尽时给出明确失败提示。

## 快速开始

1. 下载或克隆本仓库。
2. 用现代桌面浏览器打开 `index.html`。
3. 粘贴数据、拖入受支持的文本文件，或点击**加载示例**。
4. 若自动识别有误，手动选择输入格式与表头策略。
5. 点击**解析数据**。

无需服务器。仅在开发时需要本地静态服务。

## 浏览器支持

发布目标为 Windows、macOS、Linux 上最新两版 Chrome、Edge、Firefox，以及 macOS 最新版 Safari。移动端布局提供可用的降级体验，但密集的数据编辑与 JOIN 设计建议在桌面端完成。

应用使用 `localStorage`、`FileReader`、`DOMParser`、`Blob`、`TextEncoder` 和 Clipboard 事件 API 等现代浏览器 API。

## 隐私与本地存储

所有解析、筛选、JOIN 处理、复制和 Excel 生成均在浏览器内完成。应用不含任何外部资源或网络 API 调用。

默认情况下，当前工作区以键名 `ota_v20_workspace` 存储在浏览器 `localStorage` 中，刷新页面后可恢复。关闭**在此设备保存原始数据**后，源文本仅保留在当前页面会话中。**清除本地数据**按钮可移除所有已存储的工作区数据。状态栏中的存储用量条帮助你监控配额消耗。当保存失败（如配额耗尽）时，状态栏会报告错误，内存中的数据仍可用于备份。

使用敏感数据前请阅读 [PRIVACY.md](PRIVACY.md)。

## 建议规模与边界

- 单个数据源或工作区文件最大 25 MB
- 每张表默认渲染 100 行，可切换为 50、250 或 500 行
- 大数据建议关闭原始数据持久化，并在关闭页面前导出工作区备份
- 支持导出 XLSX，不支持直接读取 XLSX
- JOIN 当前使用等值条件，数据类型以解析后的表格数据为准

## 其他特性

- **暗色/亮色主题**切换，状态持久化。
- **响应式侧边栏**：视口宽度低于 760 px 时自动折叠为抽屉式面板。
- **键盘快捷键**：
  - `Ctrl`+`Enter` — 解析当前数据源
  - `Ctrl`+`N` — 新建分析页签
  - `Ctrl`+`O` — 选择本地数据文件
  - `Ctrl`+`S` — 保存当前工作区
  - `Ctrl`+`Z` / `Ctrl`+`Y` — 撤销/重做单元格编辑
  - `Ctrl`+`A` — 全选当前预览表
  - `F2` — 重命名当前页签
  - `?` — 帮助与快捷键参考
- **减弱动画**支持，响应 `prefers-reduced-motion: reduce`。
- **ARIA 语义**覆盖整个界面，包括角色、标签和动态区域，支持辅助技术。

## 开发与验证

运行时零依赖，Node.js 仅用于测试与验证。

```bash
npm test              # 运行全部 7 个测试套件
npm run build:release # 从 src/ 重建单文件发布产物
npm run validate:release  # 发布就绪检查
```

测试套件覆盖解析器格式、复制序列化、状态/存储行为、JOIN 正确性、语法检查、UI 约定、无障碍标记和离线发布约束。
