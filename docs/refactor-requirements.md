# 全面重构需求清单

本文是 Offline Table Analyzer 的重构基线，描述产品必须继续提供的能力、重构后必须满足的质量门槛，以及可以延后到后续版本的能力。它与 `docs/requirements.md` 的 v20 兼容性要求互补：v20 文档是已发布行为的事实记录，本文是重构验收清单。

## 1. 产品目标与约束

### 1.1 产品目标

Offline Table Analyzer 是一个本地优先、隐私优先的表格分析工作台。用户可以将复制文本或本地文件转换为结构化表格，进行筛选、修正、JOIN、复制和 Excel 导出。

重构的首要目标是降低单页代码的耦合和回归风险，同时保持用户可见行为、数据模型、离线分发方式和隐私边界稳定。

### 1.2 不可违反的约束

- 发布产物仍是单个可直接打开的 `index.html`。
- 运行时不依赖服务器、网络、账号、遥测、CDN 或外部脚本/样式。
- 原始数据默认只在浏览器内存和用户选择的本地存储中处理。
- v20 工作区 schema、旧版 `v16_4_store` 迁移、CLI `table-data` 输入和既有导出格式必须保持兼容。
- 不引入框架或运行时依赖，除非有可量化的性能/可维护性收益并经过单独 ADR 批准。
- 任何解析不确定性、数据截断风险、存储失败或导入失败都必须显式反馈，禁止静默破坏数据。

## 2. 用户与核心流程

### 2.1 目标用户

- 需要快速检查诊断日志、命令行表格、网页表格或复制出来的 Excel 数据的工程师。
- 处理敏感运维数据、不能上传到云端的分析人员。
- 需要把多张表临时 JOIN 后交付给其他工具的业务用户。

### 2.2 主流程

1. 打开单文件应用，创建或选择分析页签。
2. 粘贴、拖放或选择本地文本文件。
3. 自动识别格式，必要时手动切换解析器和表头模式。
4. 查看解析诊断、表格数量、列数和行数。
5. 选择表/视图、列、过滤器、亮显规则和预览方向。
6. 直接修正单元格，使用撤销/重做。
7. 设计并保存一个或多个 JOIN 视图。
8. 复制选区，或导出原始、全量、当前预览 Excel。
9. 保存/恢复工作区或仅导出配置。

## 3. 功能需求

### FR-01 工作区与页签

- 支持新建、激活、重命名、关闭、拖放排序分析页签。
- 始终保留至少一个页签；页签 ID 唯一，标题非空且唯一。
- 每个页签独立保存原始文本、解析选项、表选择、过滤规则、列选择、预览方向、分页、修正覆盖层和导出选项。
- 全局保存主题、复制格式、电子表格安全前缀策略、临时数据模式和 JOIN 视图。
- 切换页签不得串用上一个页签的 HTML 剪贴板状态、编辑历史、分页或筛选条件。
- 键盘支持新建、打开、保存、解析、重命名、关闭、全选、撤销/重做和帮助快捷键。

### FR-02 数据输入与编辑

- 支持文本粘贴、HTML 剪贴板表格、拖放文件和文件选择器。
- 支持主编辑器和全屏编辑器；两者文本、格式、表头模式和统计信息保持同步。
- 支持清空、加载示例、导出原始文本。
- 支持按扩展名预选 CSV、TSV、HTML、Markdown 等格式，同时允许用户覆盖选择。
- 输入文件和工作区文件上限为 25 MB；配置文件上限为 5 MB。
- 输入变化、文件变化、解析器变化或表头模式变化时，必须清理可能失效的行索引修正覆盖层并显示提示。

### FR-03 解析与规范化

- 保持 12 类解析器：CLI `table-data`、Data-Block、HTML、CLI 多块定宽表、ASCII、Markdown/pipe、TSV、CSV、分号 CSV、固定宽度、对齐固定宽度、空白分隔文本。
- 自动识别返回最终格式、候选格式、置信度、规范化表格和诊断集合。
- 支持 CSV 引号、转义引号、嵌入换行、HTML `rowspan`/`colspan`、Markdown 转义 pipe、BOM、CRLF、NBSP 和 `<br>`。
- 支持自动表头、强制首行表头和生成 `ColumnN` 表头；空表头和重复表头必须稳定规范化。
- 短行补空单元格；超出表头的单元格必须保留并生成 `ROW_WIDTH_MISMATCH` 诊断。
- CLI `validflag` 语义独立于通用表头推断，并支持多张表。
- 解析器不得修改用户原始文本；所有下游模块只消费统一的 normalized table contract。
- 任何格式切换都可以从诊断详情一键重解析。

### FR-04 分析、筛选和预览

- 支持全局规则、表级规则和列级 contains 筛选。
- 支持 equals、contains、not-equal、数值比较、正则、AND/OR、带引号的多词值。
- 完整正则 `/.../` 必须作为一个 token 处理，不能被 pipe OR 逻辑误拆。
- 支持亮显和“仅显示亮显”模式。
- 支持每张表可见列、折叠状态、行/列/筛选计数和 50/100/250/500 分页。
- 支持列标题模式与转置行标题模式，均支持矩形选区、自动滚动和 Ctrl/Cmd+A。
- 预览只渲染当前页 DOM，但导出必须使用完整过滤结果。
- 用户输入的表名、列名、页签名和单元格内容必须使用安全文本渲染，不能造成 HTML 注入。

### FR-05 JOIN 视图

- 支持 Inner、Left、Right、Full、Semi、Anti 六种 equality JOIN。
- 左右来源可以是原始表或已存在视图，支持链式视图。
- 支持多个等值关系、同名列自动匹配、列搜索、全选搜索结果、左右列选择、别名和拖放排序。
- 复合键使用带类型信息的 tuple 序列化，必须区分 `0`、`false`、空值和字符串值。
- 缺失关系列、非法别名、重复输出列和循环依赖必须在保存前阻止。
- 设计阶段显示匹配/未匹配估计；视图修改有未保存状态和取消确认。
- 视图可新增、复制、删除、导入和导出；导入必须进行结构安全检查。

### FR-06 修正、复制与导出

- 单元格修正以非破坏 overlay 保存，原始文本不被覆盖。
- 支持最多 100 步撤销/重做，并在来源变化时清空历史。
- 复制支持 TSV、CSV、Markdown、ASCII、Lua 单行和 Lua 展开。TSV/CSV/Markdown/ASCII 生成 HTML 表格，Lua 生成保持换行缩进的 `<pre><code>` HTML。
- Lua 使用选区中的表头作为字符串键、每条选中数据行为一个子 Table，外层索引从 `[1]` 连续递增；行头视图必须先恢复为标准字段/记录方向。
- Lua 值格式化必须支持十六进制零值归一化、非零十六进制保留、十进制数、布尔值、空字符串和 Lua 字符串转义。
- 默认防止以 `=`, `+`, `@` 或非数字 `-` 开头的内容触发电子表格公式执行；该前缀保护不应用于 Lua。
- Excel 导出支持 raw、full、preview 三种语义；full 导出支持表/视图选择和可见列投影。
- Excel 序列化必须保护长 ID、前导零、超高精度数字和非有限数字。
- 工作区备份包含完整状态；配置导出只包含规则、视图和 UI 配置，不得包含原始数据。

### FR-07 持久化、迁移与恢复

- 权威键为 `ota_v20_workspace`，schema 为 20；旧键只在 v20 成功写入后删除。
- `copyFormat` 继续使用现有全局持久化字段，允许值包括 `default`、`csv`、`markdown`、`ascii`、`lua-inline` 和 `lua-expanded`；不增加 Lua 专用持久化字段或配置界面。
- 捕获 quota/security 写入失败，显示具体状态，且内存工作区继续可用。
- 显示保存状态、时间戳、估算字节数和使用率。
- `persistRaw=false` 时序列化空 raw，但内存中的 raw 保持可用。
- 无法读取已有工作区时禁止用空默认状态静默覆盖。
- 工作区导入校验 kind、schema 前向兼容、文档数量、递归深度、数组/对象规模和危险原型键。
- 导入 ID 自动去重，标题自动唯一化；不合法 payload 必须可解释地拒绝。
- 清除本地数据必须删除 v20、legacy store 和旧输入高度键。

### FR-08 可访问性、响应式与反馈

- 主要路径为“数据输入 → 解析 → 分析 → 导出”，每一步有可见状态。
- 页签使用 tablist/tab/tabpanel 语义；重要状态使用 live region。
- 模态框关闭后恢复焦点；按钮、输入、拖放和表格编辑均有键盘路径。
- 支持 reduced-motion 和 forced-colors；移动端 760 px 以下使用侧边抽屉。
- 解析慢于 800 ms 时提示耗时；警告、错误、恢复操作均使用 Toast 或状态栏说明。

## 4. 非功能需求

- **离线性**：发布 HTML 不得包含外链、网络 API 或隐式联网。
- **隐私性**：原始数据仅进入内存、用户明确开启的 localStorage 或用户主动触发的下载。
- **安全性**：阻止 XSS、原型污染、公式注入、危险文件名和不安全工作区递归结构。
- **性能**：25 MB 输入能被拒绝或有明确反馈；预览渲染受分页约束；重复解析不应产生无界 DOM。
- **兼容性**：支持最新两版 Chrome、Edge、Firefox，以及最新 Safari 桌面版；Node 校验环境为 20+。
- **可维护性**：业务职责按模块分离，模块依赖方向明确，发布产物由确定性构建生成。
- **可测试性**：领域逻辑可在无真实浏览器环境运行；UI 合同可静态验证；关键流程有浏览器回归测试扩展点。
- **可回退性**：每个迁移阶段可单独提交、验证和回退；生成的 `index.html` 不作为手工编辑源。

## 5. 稳定数据契约

### NormalizedTable

```text
{
  name: string,
  headers: string[],
  rows: string[][],
  sourceType: string,
  meta: object,
  diagnostics: Diagnostic[]
}
```

### Diagnostic

```text
{
  code: string,
  severity: "info" | "warning" | "error",
  message: string,
  table?: string,
  row?: number,
  column?: number,
  details?: object
}
```

### 不变量

- `headers.length` 等于每一行的逻辑宽度；缺失值是空字符串，溢出值不能丢失。
- `headers` 在单表内唯一且非空。
- `rows` 保持解析顺序；过滤和分页不得修改原始行顺序。
- JOIN 视图名称全局唯一，依赖图无环。
- 所有可持久化 ID、标题和规则都能经过 normalize 后再次序列化/反序列化。

## 6. 测试验收矩阵

| 层级 | 必须覆盖 | 验收方式 |
| --- | --- | --- |
| 领域单元 | 表格规范化、表头推断、12 类解析器、筛选 token、JOIN、复制/Excel 序列化 | Node 测试，覆盖正常、边界、恶意输入 |
| 状态单元 | 默认状态、迁移、去重、临时数据、配额失败、撤销/重做 | Node + VM 隔离存储测试 |
| 模块集成 | ImportEngine → normalized table、Store → App、Joiner → Preview、Exporter | 公开契约测试和构建后脚本测试 |
| UI 合同 | 关键 ID、ARIA、快捷键、回归方法、离线约束 | 静态契约 + JS 语法校验 |
| 浏览器回归 | 粘贴 → 解析 → 筛选 → JOIN → 复制 → 导出、移动端、主题、文件导入 | CI 可选 headless 浏览器矩阵 |
| 发布 | 单 HTML、无外链、版本一致、构建可重复 | `npm run validate:release` |

## 7. 完成定义

一次重构阶段只有同时满足以下条件才算完成：

- 需求/架构决策已经记录，变更范围和回退方式明确。
- 新旧行为契约测试均通过，新增代码有对应测试或静态验收说明。
- `npm test`、`npm run validate:release` 和 `npm run build:release` 全部通过。
- 生成的 `index.html` 可在无服务器环境直接打开，且无外部资源。
- 没有把测试 hook、调试数据或原始用户数据带入生产发布。
- 工作区状态干净，提交粒度足以单独回退。
