# 自动识别优化计划

## 目标

提升自动格式识别、自动表头推断和数据完整性保护，重点解决“格式分数高但解析结果为空”“错误格式改变列结构”“定宽解析静默截断”三类问题。

## 已实施的第一阶段

1. 分隔符识别必须有真实的分隔符证据；无逗号文本不再获得 CSV 兜底分。
2. CSV/TSV 的分隔符统计复用引号感知解析，避免引号字段中的 Tab 干扰格式判断。
3. ASCII 表格要求至少两个强边框行和至少两行数据，普通 `---` 不再触发。
4. 定宽对齐格式要求同时存在可识别的多列头行和数据行。
5. 自动识别会对候选执行轻量解析质量校验；候选无法生成表格时自动尝试后续候选。
6. 表头阈值从 0.55 提升到 0.65，增加中英文常见字段名，并返回表头置信度和判断理由。
7. 定宽对齐数据行发现位置偏移时，优先使用空白分隔结果保留溢出值，并发出 `ALIGNED_POSITION_MISMATCH` 诊断。
8. 新增空白文本、引号内 Tab、原始 HTML、ASCII 误判、歧义字符串行和定宽溢出的回归测试。
9. 定宽对齐解析改为独立的 `TextLayout` 推断：以数据行的稳定列起点为主证据，结合表头语义和类型差异判断“列头还是表名”，支持多词列头和多词数据单元格。

## 第二阶段建议

后续可将识别流程升级为“特征提取 → 候选解析 → 解析质量评分 → 歧义提示”的统一管线，增加以下质量指标：行列矩形度、分隔符覆盖率、类型稳定性、类型/词法表头证据、位置支持率、诊断惩罚、截断风险和候选分数差距。对于最高两个候选分数接近的输入，应在界面中明确标记为“需要确认”，而不是伪装成确定的百分比。

## 验收标准

- 标准 CSV、TSV、HTML、Markdown、ASCII、CLI、固定宽度和定宽对齐样例保持通过。
- 无分隔符的空白文本不再被识别为 CSV。
- 合法 CSV 引号字段中的 Tab 不改变格式判断。
- 纯水平线不再产生空的定宽对齐结果。
- 定宽对齐值发生位置偏移时不静默丢失内容，并有诊断信息。
- 构建产物、全量回归测试和发布校验全部通过。
